Zeit messen

Zum Messen der Zeit gibt es grundsätzlich zwei Methoden, je nachdem was genau gemessen werden soll:

  • 1. Zeit im Verhältnis zur Programmgeschwindigkeit

    Die Zeit wird in Abhänigkeit der gezeichneten Bilder (FPS - Frames per second) gemessen. Läuft das Programm also langsamer z.B. auf einem langsamen Computer, so läuft die Zeit auch langsamer, da sie nicht an die Echtzeit, sondern an die Programmgeschwindigkeit gebunden ist.

    Hinweis: Diese Anleitung geht davon aus, dass du das Grund-Programm übernommen und somit eine Variable FPS mit der angestrebten Anzahl Bilder pro Sekunde gesetzt hast.

    Um das zu erreichen benötigst du eine Variable in der die Anzahl Frames (Bilder) seit Programmstart gespeichert werden. Dazu gehst du wie folgt vor:

    1. Definiere die Variable ausserhalb der Haupt-Schleife.
    2. Wenn das Programm startet, setze die Variable auf 0.
    3. In jedem Schleifendurchlauf der Haupt-Schleife addiere eins zu der Variable.
    Damit ist die Grundlage gelegt um die Laufzeit anzuzeigen. Um diese zu berechnen musst du die gerade erstellte Variable lediglich noch durch FPS teilen. Damit erhälst du Sekunden seit dem Starten der Zählung.

    Wie du einen Text anzeigen kannst findest du auf hier.

  • 2. Echtzeit, unabhängig davon wie schnell das Programm läuft

    Die verstrichene Zeit lässt sich messen, indem man sich die Zeit zu Beginn des Programms merkt. Damit kann man zu jedem späteren Zeitpunkt eine Differenz bilden und so die verstrichene Zeit berechnen. Pygame had dazu eingene Befehle. Die Dokumentation dazu findest du hier. Alternativ kannst du dazu aber auch die Bibliothek time verwenden. (Vollständige time Dokumentation)

    Ein sehr simples Beispielprogramm könnte also so aussehen:

    
import time
start_zeit = time.time()

# Berechne irgend etwas ...

end_zeit = time.time()
differenz = end_zeit - start_zeit
print("Zeit:", differenz) # Verstrichene Zeit in Sekunden
